# HayaiTTS

A 100% offline Android text-to-speech engine that registers itself as a
system-wide TTS provider, so any app that talks via Android's standard
`TextToSpeech` API can use it. The companion to [Hayai](https://github.com/HayaiApp/hayai),
focused on bringing neural voices (Piper, VITS, Matcha, Kokoro, Kitten) to
phones without phoning home.

## What ships today

- **System-wide TTS engine** — appears in *Settings → System → Languages & input
  → Text-to-speech engines*. Pick HayaiTTS, set it as the preferred engine,
  and every app on the device that calls `tts.speak(...)` now uses neural
  voices instead of the stock Google or Samsung engine.
- **Five model families**, all running on-device via [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx)
  + ONNX Runtime:
  - **Piper** — fast, small, MIT-licensed; the default low-resource pick.
  - **VITS** — high quality multi-speaker (English VCTK, Chinese AISHELL3).
  - **Matcha-TTS** — diffusion-based, very high quality (English LJSpeech,
    Chinese Baker).
  - **Kokoro 82M** — multilingual flagship (English / Chinese / Japanese / 50+
    languages).
  - **Kitten Nano** — the smallest neural English voice (~30 MB).
- **Material 3 Expressive UI** — `MaterialExpressiveTheme` + `MotionScheme.expressive()`
  + the wavy progress / shape morphing / spring motion lineup. The bundled
  voice (Piper Amy en\_US-low) speaks the moment you install the app; everything
  else streams on demand from the [HayaiApp/HayaiTTS catalog](catalog/v1/models.json).
- **Custom model import** — pick a sherpa-onnx-compatible `.tar.bz2`, `.zip`, or
  bare `.onnx` from device storage. The analyzer detects the family (Piper /
  VITS / Matcha) by file layout, lets you confirm or override, and installs
  the bundle into the engine's voice library.
- **Per-locale defaults** — set a different voice for English vs. Japanese vs.
  Chinese; the TTS service routes synthesis requests to the right model based
  on the locale Android passes in.
- **Wi-Fi-only downloads**, internal-vs-external storage location, R8-minified
  release builds, ABI-split APKs.

## Build

Requires JDK 21 (the JBR shipped with Android Studio works; Temurin works
too).

```bash
git clone https://github.com/HayaiApp/HayaiTTS.git
cd HayaiTTS
./gradlew assembleDebug
```

On Windows pass the JDK explicitly so Gradle doesn't pick up the JRE from
`PATH`:

```powershell
.\gradlew assembleDebug `
  -Dorg.gradle.java.home="C:\Program Files\Android\Android Studio\jbr"
```

Output: `app/build/outputs/apk/debug/app-debug.apk` (~145 MB — most of that
is the bundled Piper Amy voice + sherpa-onnx native libraries across three
ABIs).

For a release-mode build (R8, ABI splits, signed APK), see
[ONBOARDING.md](ONBOARDING.md#release-signing).

## Catalog

The voice catalog is generated by [`tools/catalog/build_catalog.py`](tools/catalog/build_catalog.py),
which scrapes the [sherpa-onnx TTS model index](https://k2-fsa.github.io/sherpa/onnx/tts/all/),
HEAD-and-streams every release tarball to compute a sha256, and emits a
deterministic `catalog/v1/models.json`. A weekly GitHub Action re-runs the
script and opens a PR if any voice has appeared, vanished, or been rebuilt
upstream.

Custom voices that aren't on the upstream index can be sideloaded via the
Library → Import flow.
